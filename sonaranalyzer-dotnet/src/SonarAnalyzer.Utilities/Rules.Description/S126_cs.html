此规则应用于一条 if 语句后跟有一条或多条 else if 语句的情况，最后的 else if语句后应跟有 else 语句。

这种对结尾的 else 语句的要求属于防御性的编程（defensive programming）。

这种 else 语句里要么实现一个相应的功能，要么包含一条适当的注释来说明没有在此处实现功能的原因。这与必须要有一条 default 分句在 switch 结尾的要求是一致的。

不合规的代码示例

if (x == 0)
{
  doSomething();
} else if (x == 1)
{
  doSomethingElse();
}

合规的解决方案

if (x == 0)
{
  doSomething();
} else if (x == 1)
{
  doSomethingElse();
} else
{
  throw new IllegalStateException();
}

  • MISRA C++:2008, 6-4-2 - All if...else if constructs shall be terminated with an else clause.
  • MISRA C:2012, 15.7 - All if...else if constructs shall be terminated with an else statement
  • CERT, MSC01-C. - Strive for logical completeness
  • CERT, MSC57-J. - Strive for logical completeness
