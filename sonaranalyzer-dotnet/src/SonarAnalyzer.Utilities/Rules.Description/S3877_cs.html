有些方法应该谨慎地调用，但这些方法之外，比如 ToString 方法，只要能正常工作即可。在这样的方法里抛出异常很可能会意外地中断调用者的代码。

在以下场合抛出异常时会引发问题：

  • 事件访问器
  • Object.Equals
  • IEquatable.Equals
  • GetHashCode
  • ToString
  • static 型构造函数
  • IDisposable.Dispose
  • operator ==, !=, <, >, <=, >=
  • implicit 强制转换运算符

public override string ToString()
{
  if (string.IsNullOrEmpty(Name))
  {
    throw new ArgumentException("...");  // 不合规
  }
  //……

例外

System.NotImplementedException 及其派生类被此规则忽略。

System.InvalidOperationException，System.NotSupportedException 和 System.ArgumentException 以及它们的派生类在事件访问器中被此规则忽略。
