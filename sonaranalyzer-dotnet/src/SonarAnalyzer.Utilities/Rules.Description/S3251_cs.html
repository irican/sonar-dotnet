partial 方法使得编写系统有了更强的灵活性。通过调用定义了Hook的特征的方法，可以将它们加到生成的代码中，但这样的写法可能最终都还没有将其完全实现。如果在代码投入生产时还没有完全实现，编译器会默认移除那些对它的调用。最好的情况是这些调用只会让代码显得很繁琐，但最坏的情况下它们会很严重，功能不完全会导致在运行时出现意外的结果。

此规则会对在程序集中找不到实现的partial方法引发问题。

不合规的代码示例

partial class C
{
  partial void M(); //不合规

  void OtherM()
  {
    M(); //不合规，会被清除
  }
}
