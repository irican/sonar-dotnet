计数器变化方向错误的 for 并不是无限循环。由于环绕（wraparound）原因，循环最终会停止，如果这样做了，循环运行次数会远远多于预期，导致非预期行为。

不合规的代码示例

public void DoSomething(string[] strings)
{
  for (int i = 0; i < strings.Length; i--)  // 不合规
  {
    string s = strings[i];  // i的值到达-1时会抛出IndexOutOfRangeException
    // 做点别的
  }
}

合规的解决方案

public void DoSomething(string[] strings)
{
  for (int i = 0; i < strings.Length; i++)
  {
    string s = strings[i];
    // 做点别的
  }
}

请参阅

  • CERT, MSC54-J. - Avoid inadvertent wrapping of loop counters
