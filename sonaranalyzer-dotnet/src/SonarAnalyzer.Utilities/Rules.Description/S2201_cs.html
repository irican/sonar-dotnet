当某个函数的调用没有任何其他的附带效果时，如果忽略函数的结果，那么调用它的意义在哪里呢？出现了这种调用时，要么这是一次无用的调用，应该将其删除；要么说明源代码并没有如预期一样运行。

以下方法的结果被忽略时不会引发问题：

  • LINQ 方法
  • [Pure] 方法，
  • 任何 string， int，……， System.Collections.Immutable.ImmutableArray<T>， ImmutableHashSet<T> 等类型的方法

不合规的代码示例

coll.Where(i => i > 5).Select(i => i*i); // 不合规
"this string".Equals("other string"); // 不合规

合规的解决方案

var res = coll.Where(i => i > 5).Select(i => i*i);
var isEqual = "this string".Equals("other string");

例外

对于有参数 out 和 ref 的方法调用，此规则不会报告问题。

请参阅

  • MISRA C:2012, 17.7 - The value returned by a function having non-void return type shall be used
  • CERT, EXP12-C. - Do not ignore values returned by functions
  • CERT, EXP00-J. - Do not ignore values returned by methods
