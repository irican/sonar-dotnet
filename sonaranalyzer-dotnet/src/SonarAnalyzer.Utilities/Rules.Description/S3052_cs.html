编译器在将类的字段、自动属性（auto-properties）和事件设为初始化值（initialization values）之前，会自动先将它们初始化为默认值，因此没有必要再显式地将他们赋值为默认值。而且，在“简洁的代码是更好的代码”的原则下，这样的行为会被认为是不好的代码风格。

不合规的代码示例

class X
{
  public int field = 0; // 不合规
  public object o = null; // 不合规
  public object MyProperty { get; set; } = null; // 不合规
  public event EventHandler MyEvent = null;  // 不合规
}

合规的解决方案

class X
{
  public int field;
  public object o;
  public object MyProperty { get; set; }
  public event EventHandler MyEvent;
}

例外

此规则忽略 const 字段。
