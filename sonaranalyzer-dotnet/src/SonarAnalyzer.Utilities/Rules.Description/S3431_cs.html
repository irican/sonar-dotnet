即使是粗略地阅读代码也应该明确被测试的代码是哪一部分，以及预期的结果是什么。不幸的是，特性 [ExpectedException]并不是这样，因为方法中的任何位置都可能抛出异常。

此方法会检查是否使用了MSTest和NUnit的 ExpectedException 属性。

不合规的代码示例

[TestMethod]
[ExpectedException(typeof(ArgumentNullException))]  // 不合规
public void TestNullArg()
{
  //……
}

合规的解决方案

[TestMethod]
public void TestNullArg()
{
  bool callFailed = false;
  try
  {
    //……
  }
  catch (ArgumentNullException)
  {
    callFailed = true;
  }
  Assert.IsTrue(callFailed, "Expected call to MyMethod to fail with ArgumentNullException");
}

例外

此规则忽略只有一行的测试方法，因为这样的方法显然就是用来抛出异常的。
