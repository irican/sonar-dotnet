可以使用匿名的委托来订阅事件（subscribe to events），但这样做之后就不可能再取消订阅。这是因为订阅的过程实际上是将委托添加到一个列表中。而取消订阅本质上就是从订阅列表里移除 这一项。但如果使用了匿名委托，取消订阅操作会试图移除与所添加项目不同的项目。结果只能是：NOOP。

相反地，应将委托存入一个变量，然后使用这个变量来订阅事件和取消订阅。

不合规的代码示例

listView.PreviewTextInput += (obj,args) =>
        listView_PreviewTextInput(obj,args,listView);

// ……

listView.PreviewTextInput -= (obj, args) =>
        listView_PreviewTextInput(obj, args, listView); // 不合规，这个委托不会被订阅

合规的解决方案

EventHandler func = (obj,args) => listView_PreviewTextInput(obj,args,listView);

listView.PreviewTextInput += func;

// ……

listView.PreviewTextInput -= func;
