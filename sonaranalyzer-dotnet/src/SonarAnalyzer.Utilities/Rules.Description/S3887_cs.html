对字段使用 readonly 关键字意味着该字段初始化后不能再被更改。然而，在对集合或数组使用时，这句话只是部分正确。readonly 会禁止将另一个实例赋值给被修饰的字段，但无法阻止该字段内部的内容被更新。这意味着字段的值事实上是可以修改的，对这样的字段使用 readonly 是有误导性的，您可能无法进行预期的行为。

此规则会在遇到非private的 readonly 数列或集合时引发问题。

不合规的代码示例

public class MyClass
{
  public readonly string[] strings;  // 不合规

  // ……

合规的解决方案

public class MyClass
{
  public string[] strings;

  // ……

或者

public class MyClass
{
  private readonly string[] strings;

  // ……
