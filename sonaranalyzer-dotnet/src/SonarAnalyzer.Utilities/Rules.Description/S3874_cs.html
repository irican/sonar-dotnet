通过引用传递参数（通常发生在你使用 out 或 ref 修饰符时）意味着方法会接收到指向参数的指针，而不是参数自身。如果该参数是值类型，则此方法可以更改该参数的值；如果参数是引用类型，那么此方法得到的是指向指针的指针，而这通常来说并不符合人们的预期。即使使用者知道这一点并有意这样做，这也是一件很难纠正的事，应该在使用时谨慎小心。

当修饰符 out 或 ref 用于public方法的非 Optional 参数时此规则会引发一个问题。另外，Optional 参数由规则S3447检查。

不合规的代码示例

public void GetReply(
         ref MyClass input, // 不合规
         out string reply)  // 不合规
{ ... }

合规的解决方案

public string GetReply(MyClass input)
{ ... }

public bool TryGetReply(MyClass input, out string reply)
{ ... }

public ReplyData GetReply(MyClass input)
{ ... }

internal void GetReply(ref MyClass input, out string reply)
{ ... }

例外

下列方法不会引发此问题：

- 非public类型的方法

- 只有‘out’参数，名字以“Try”开头并返回bool值的方法

- 实现接口的方法
