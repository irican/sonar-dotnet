字段不应成为API的一部分，因此应始终是private型的。例如，它们实际上无法添加到接口中，并且在以后如不破坏其向后兼容性的情况下是无法添加验证（validation）的。因此，开发者反而应将他们的字段封装在属性（property）中。显式的属性getter方法和属性setter方法可以用于验证，也可以实现向新系统的平滑过渡。

不合规的代码示例

public class Foo
{
  public int MagicNumber = 42;
}

合规的解决方案

public class Foo
{
  public int MagicNumber
  {
    get { return 42; }
  }
}

或者 

public class Foo
{
  private int MagicNumber = 42;
}

例外

此规则忽略 struct，类中的 static 和 const 字段也是一样。

另外，考虑类的可访问性的话，只有其真正的可访问性是 public 时才会引发问题。
