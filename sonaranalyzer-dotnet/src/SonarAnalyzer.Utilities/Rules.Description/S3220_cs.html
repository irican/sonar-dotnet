方法解析的规则很复杂，可能不是所有的编程人员都能正确地理解。params 关键字能让方法的声明以不明显的方式重叠，这样一来所调用参数的类型上的轻微差别都会使该方法解析为不同的方法。

当某次方法调用被解析为带 params 的方法的声明时，此规则会引发问题。但这次调用也可能会解析到另一个不带 params 的方法。

不合规的代码示例

public class MyClass
{
  private void Format(string a, params object[] b) { }

  private void Format(object a, object b, object c) { }
}

// ...
MyClass myClass = new MyClass();

myClass.Format("", null, null); //不合规，这次调用会解析到第一个带param的Format方法，但这是开发者的本意吗？
