在方法主体内给参数赋值尽管从技术角度来说是没有错的，但如果尚未读取过该参数，这样做有可能产生bug。因此，参数的初值、捕获的异常和foreach语句的参数如果不是 final ，那么至少在将他们读取之后再进行重新赋值。

不合规的代码示例

public void DoTheThing(string str, int i, List<string> strings)
{
  str = i.ToString(i);  // 不合规

  foreach (var s in strings)
  {
    s = "hello world";  // 不合规
  }
}
