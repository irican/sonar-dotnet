重复使用的字符串常量会让进程在重构时易于出错，因为它出现的每一处都需要更改。

而使用constant变量的话，你可以在多处引用它的值而只需在一处修改。

不合规的代码示例

public class Foo
{
    private string name = "foobar"; // 不合规

    public string DefaultName { get; } = "foobar"; // 不合规

    public Foo(string value = "foobar") // 不合规
    {
        var something = value ?? "foobar"; // 不合规
    }
}

合规的解决方案

public class Foo
{
    private const string Foobar = "foobar";

    private string name = Foobar;

    public string DefaultName { get; } = Foobar;

    public Foo(string value = Foobar)
    {
        var something = value ?? Foobar;
    }
}

例外

以下情况可忽略：

  • 少于五个字符的字符串常量
  • 与参数名称匹配的字符串常量
  • 在特性（attribute）中使用的字符串常量
