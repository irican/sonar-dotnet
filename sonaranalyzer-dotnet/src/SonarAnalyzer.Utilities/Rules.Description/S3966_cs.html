正确的 IDisposable.Dispose 实现应允许它在同一个对象上调用多次，然而这并不一定会发生，而且这种情况还可能会导致有异常抛出。

但最好不要依赖这种正确的实现，应确保对象在它所有的可执行路径上都只被清理一次。尤其是在处理嵌套的 using 语句时。

不合规的代码示例

using (Stream stream = new FileStream("file.txt", FileMode.OpenOrCreate))
{
    using (StreamWriter writer = new StreamWriter(stream))  // 不合规：‘stream’会被清理两次
    {
        // 使用对象writer
    }
}

合规的解决方案

Stream stream = null;
try
{
    stream = new FileStream("file.txt", FileMode.OpenOrCreate);
    using (StreamWriter writer = new StreamWriter(stream))
    {
        stream = null;
        // 使用对象writer
    }
}
finally
{
    if(stream != null)
        stream.Dispose();
}
