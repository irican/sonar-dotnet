调用者信息属性（CallerFilePathAttribute，CallerLineNumberAttribute 和 CallerMemberNameAttribute）提供了通过可选参数获得方法的调用者的信息的途径。但只有在非显式地提供它们的值时，它们才能正常工作。因此如果你在参数列表的中间定义了一个带有调用者信息属性的方法，那么你就将该方法的调用者摆在了一个很不好的境地：如果它们想正确地使用方法，就必须使用命名的参数（named argument）。

不合规的代码示例

void TraceMessage([CallerMemberName] string memberName = "",
  [CallerFilePath] string filePath = "",
  [CallerLineNumber] int lineNumber = 0,
  string message = null)  // 不合规
{
  /* …… */
}

合规的解决方案

void TraceMessage(string message = null,
  [CallerMemberName] string memberName = "",
  [CallerFilePath] string filePath = "",
  [CallerLineNumber] int lineNumber = 0)
{
  /* …… */
}
