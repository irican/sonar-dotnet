public类中的public字段不符合封装的原则，并且有以下三个主要的缺点：

  • 之后无法增添其他的功能，如验证（validation）。
  • 字段在类内部的表示是对外暴露的，并且之后无法更改。
  • 类成员的值可在代码的任何一处被修改，可能会出现编程者预料不到的结果。

使用private字段和public属性（set与get）可制止未授权的修改。属性还有额外的安全特性，如Link Demands。

注意，由于对简单属性的优化，因此选用public字段对性能的提升其实十分有限。

不合规的代码示例

public class Foo
{
    public int instanceData = 32; // 不合规
}

合规的解决方案

public class Foo
{
    private int instanceData = 32;

    public int InstanceData
    {
        get { return instanceData; }
	set { instanceData = value ; }
    }
}

例外

此规则忽略标有 readonly 或 const 的字段。

此规则忽略注有 StructLayoutAttribute 的类或结构体中的字段。
