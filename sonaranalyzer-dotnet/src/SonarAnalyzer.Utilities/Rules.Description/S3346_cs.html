断言（assertion）是在开发过程中的编译调试模式下使用的一段代码。它允许一个程序在运行时检查自身。断言为true意味着一切都在按照预期运行。

在非调试模式下，所有的 Debug.Assert 都会自动删除。因此，根据使用约定，由这些断言判断的布尔值表达式绝对不能含有任何附带的其他功能（side effect）。否则，在退出调试模式时，应用程序的功能行为会不再与之前相同。

以下列的任何一项开头的方法名都会触发此规则：remove，delete，add，pop，update，retain，insert，push，append，clear，dequeue，enqueue，dispose，put 或 set。SetEquals会被忽略。

不合规的代码示例

Debug.Assert(list.Remove("dog"));

合规的解决方案

bool result = list.Remove("dog");
Debug.Assert(result);
