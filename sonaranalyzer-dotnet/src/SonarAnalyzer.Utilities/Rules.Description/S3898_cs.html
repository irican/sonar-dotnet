如果你正在使用 struct，很可能是因为你对性能感兴趣。但如果不实现 IEquatable<T>，在你进行比较时性能就会下降。因为没有 IEquatable<T> 时，为做比较会进行装箱（boxing）和反射（reflection）操作。

不合规的代码示例

struct MyStruct  // 不合规
{
  private int i;
  public int I
  {
    //……
  }
}

合规的解决方案

struct MyStruct : IEquatable<MyStruct>
{
  private int i;
  public int I
  {
    //……
  }

  public bool Equals(MyStruct other)
  {
    throw new NotImplementedException();
  }
}
