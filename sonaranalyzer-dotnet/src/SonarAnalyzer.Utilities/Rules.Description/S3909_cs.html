NET Framework 2.0 引入了泛型接口 System.Collections.Generic.IEnumerable<T>，比起较旧的、非泛型的接口，应该优先考虑使用泛型接口。

当一个public类型实现了 System.Collections.IEnumerable 时此规则会引发一个问题。

不合规的代码示例

using System;
using System.Collections;

public class MyData
{
  public MyData()
  {
  }
}

public class MyList : CollectionBase // 不合规
{
  public void Add(MyData data)
  {
    InnerList.Add(data);
  }

  // ……
}

合规的解决方案

using System;
using System.Collections.ObjectModel;

public class MyData
{
  public MyData()
  {
  }
}

public class MyList : Collection<MyData>
{
  // 此处进行具体的实现
}
