一个 for 循环的停止条件（stop condition）应当是将循环次数的计数器（loop counter）与一个恒定的值进行比较(即在每一次循环的始末该条件的值都为真)。理想的情况是在即将进入循环结构时，让停止条件与一个局部变量的值联系起来。

非恒定的循环停止条件会让效率有所降低，会让代码的理解和维护变得困难，还很可能在将来引入错误。

此规则监测三种非恒定的停止条件：

  • 循环的计数器在 for 循环内被更新
  • 停止条件依赖于一个方法调用
  • 停止条件依赖于一个对象属性。因为这样的属性可能随着循环的执行而改变

不合规的代码示例

class Foo
{
    static void Main()
    {
        for (int i = 1; i <= 5; i++)
        {
            Console.WriteLine(i);
            if (condition)
            {
               i = 20;
           }
        }
    }
}

合规的解决方案

class Foo
{
    static void Main()
    {
        for (int i = 1; i <= 5; i++)
        {
            Console.WriteLine(i);
        }
    }
}

请参阅

  • MISRA C:2004, 13.6 - Numeric variables being used within a for loop for iteration counting shall not be modified in the body of the loop.
  • MISRA C++:2008, 6-5-3 - The loop-counter shall not be modified within condition or statement.
