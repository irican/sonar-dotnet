不必要的关键字只会让代码变得杂乱，应该将其删除。所谓不必要具体指的是：

  • 在某处已经完整的类型声明里使用 partial
  • 给 sealed 类里的成员使用 sealed
  • 在已经标有 unsafe 的结构（construct）里，或是块中没有 unsafe的结构时，仍给方法或语句块使用了 unsafe
  • 对没有整型算术运算的语句块使用 checked 和 unchecked

不合规的代码示例

public partial class MyClass // 不合规
{
  public virtual void Method()
  {
  }
}

public sealed class MyOtherClass : MyClass
{
  public sealed override void Method() // 不合规
  {
  }
}

合规的解决方案

public class MyClass
{
  public virtual void Method()
  {
  }
}

public sealed class MyOtherClass : MyClass
{
  public override void Method()
  {
  }
}
