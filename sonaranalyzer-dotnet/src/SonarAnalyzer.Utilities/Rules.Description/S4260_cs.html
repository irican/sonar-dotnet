在创建接受WPF参数的自定义标记扩展时，必须使用ConstructorArgument标记来标识匹配这些参数的离散属性。但是，由于这是通过字符串完成的，编译器不会注意到是否有拼写错误。

当ConstructorArgumentAttribute 的字符串参数不匹配任何构造函数的任何参数时，该规则会引发一个问题。

不合规的代码示例

using System;

namespace myLibrary
{
  public class MyExtension : MarkupExtension
  {
    public MyExtension() { }

    public MyExtension(object value1)
    {
      Value1 = value1;
    }

    [ConstructorArgument("value2")]
    public object Value1 { get; set; }
  }
}

合规的解决方案

using System;

namespace myLibrary
{
  public class MyExtension : MarkupExtension
  {
    public MyExtension() { }

    public MyExtension(object value1)
    {
      Value1 = value1;
    }

    [ConstructorArgument("value1")]
    public object Value1 { get; set; }
  }
}
