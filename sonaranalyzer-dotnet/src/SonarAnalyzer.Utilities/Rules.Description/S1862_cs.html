一连串的 if/else if 语句会被从上到下地进行评估。然而最多只有一条分支会被执行：即第一个条件判断为true 的分支。

因此，重复的条件会自动形成死代码。通常这是由于复制／粘贴的错误造成的。最好的情况下，这种行为只会产生伪代码。但是最坏的情况下它会形成一个bug，而且在维护时很可能会引发更多的bug。另外，它显然还可能会导致意外的行为。

不合规的代码示例

if (param == 1)
{
  OpenWindow();
}
else if (param == 2)
{
  CloseWindow();
}
else if (param == 1) // 不合规
{
  MoveWindowToTheBackground();
}

合规的解决方案

if (param == 1)
{
  OpenWindow();
}
else if (param == 2)
{
  CloseWindow();
}
else if (param == 3)
{
  MoveWindowToTheBackground();
}

请参阅

  • CERT, MSC12-C. - Detect and remove code that has no effect or is never executed
