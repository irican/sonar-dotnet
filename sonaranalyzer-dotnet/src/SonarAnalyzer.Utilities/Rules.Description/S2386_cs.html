可直接访问的类所拥有的 public static 可变字段应尽可能的保护起来。可以通过减弱字段的可访问性，或者将返回值改为不可变的类型来实现这一点。

在出现继承／实现了System.Array 或 System.Collections.Generic.ICollection<T> 的public static 字段时，此规则会引发问题。

不合规的代码示例

public class A
{
  public static string[] strings1 = {"first","second"};  // 不合规
  public static List<String> strings3 = new List<String>();  // 不合规
  // ……
}

合规的解决方案

public class A
{
  protected static string[] strings1 = {"first","second"};
  protected static List<String> strings3 = new List<String>();
  // ……
}

例外

以下情况不会引发问题：

  • 如果字段的类型是继承／实现了以下一个（或多个）类型：
    • System.Collections.ObjectModel.ReadOnlyCollection<T>
    • System.Collections.ObjectModel.ReadOnlyDictionary<TKey, TValue>
    • System.Collections.Immutable.IImmutableArray<T>
    • System.Collections.Immutable.IImmutableDictionary<TKey, TValue>
    • System.Collections.Immutable.IImmutableList<T>
    • System.Collections.Immutable.IImmutableSet<T>
    • System.Collections.Immutable.IImmutableStack<T>
    • System.Collections.Immutable.IImmutableQueue<T>
  • 如果字段是 readonly 型，并且是以不可变的类型（即继承／实现了上表中的一个类型）或null值内联初始化的。
