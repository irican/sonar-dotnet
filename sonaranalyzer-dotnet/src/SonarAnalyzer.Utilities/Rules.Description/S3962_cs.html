static readonly 字段的值是在运行时计算的，const 字段的值则是在编译时计算，这样的安排能提高性能。

当一个 static readonly 字段被编译时可计算的数值初始化时，此规则会引发一个问题。

Microsoft指定的可拥有常数值的类型列表如下：

C# type .Net Fwk type
bool System.Boolean
byte System.Byte
sbyte System.SByte
char System.Char
decimal System.Decimal
double System.Double
float System.Single
int System.Int32
uint System.UInt32
long System.Int64
ulong System.UInt64
short System.Int16
ushort System.UInt16
string System.String

不合规的代码示例

namespace myLib
{
  public class Foo
  {
    static readonly int x = 1;  // 不合规
    static readonly int y = x + 4; // 不合规
    static readonly string s = "Bar";  // 不合规
  }
}

合规的解决方案

namespace myLib
{
  public class Foo
  {
    const int x = 1;
    const int y = x + 4;
    const string s = "Bar";
  }
}
