NaN 并不等于任何数，甚至是它自己。与 NaN 进行相等或不等的比较，其结果是可以预见的，但很可能不是你想要的。

相反地，判断一个变量是否与 NaN 相等的最好方法是使用 Number.isNaN()。自ES2015后，还可以让它与自己比较（虽然可能有点违反常识）。因为既然 NaN !== NaN，那么当 a !== a时，你就能认为它等于 NaN。

不合规的代码示例

var a = double.NaN;

if (a == double.NaN) // 不合规；值总为假
{
  Console.WriteLine("a is not a number");  // 死代码
}
if (a != double.NaN)  // 不合规；值总为真
{
  Console.WriteLine("a is not NaN"); // 该语句的意思不一定正确
}

合规的解决方案

if (double.IsNaN(a))
{
  console.log("a is not a number");
}
