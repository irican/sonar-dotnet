用你即将赋给变量的值来检查变量是毫无意义的，省下这些循环和代码直接赋值即可。

不合规的代码示例

if (x != a)  // 不合规，何必呢
{
  x = a;
}

合规的解决方案

x = a;

例外

此规则对属性不做检查，因为它们可能有附带功能，移除这样的检查可能会导致意外的附带功能。

if (MyProperty != a)
{
    MyProperty = a; // 合规，因为调用setter可能会产生很大开销
}

private int myField;
public int SomeProperty
{
    get
    {
        return myField;
    }
    set
    {
        if (myField != value)
        {
            myField = value;
        }
    }
}
