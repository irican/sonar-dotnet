一个局部变量被赋值后没有任何指令读取该值，就会出现无用存储（dead store）。计算或检索一个值后直接将其覆盖或丢弃，表明代码中可能有严重的错误。纵使不是错误，该行为也是资源的浪费。所以，所有计算后的值都应该用到。

不合规的代码示例

i = a + b; // 不合规；计算结果在覆盖前并没有用过
i = compute();

合规的解决方案

i = a + b;
i += compute();

例外

以下情况不作报告：

  • 方法主体中有 try 代码块，
  • lambda 表达式捕获局部变量，或者
  • 有变量未被使用 (规则S1481覆盖此情况)，
  • 初始化为 -1，0，1，null，true，false，""和 string.Empty。

请参阅

  • MITRE, CWE-563 - Assignment to Variable without Use ('Unused Variable')
  • CERT, MSC13-C. - Detect and remove unused values
  • CERT, MSC56-J. - Detect and remove superfluous code and values
