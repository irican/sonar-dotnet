Obsolete 特性在应用时可以不带参数，但如果你真的这样使用了 Obsolete ，你就没有说明废弃（obsolete）该部分的原因，也没有给出建议的替代项，这让维护者在每次看到警告后（warning）都需要再花时间把这些情况搞清楚。

不合规的代码示例

public class Car
{

  [Obsolete]  // 不合规
  public void CrankEngine(int turnsOfCrank)
  { ... }
}

合规的解决方案

public class Car
{

  [Obsolete("Replaced by the automatic starter")]
  public void CrankEngine(int turnsOfCrank)
  { ... }
}
