如果某个方法标记了下列特性中的一个就会被检测为测试方法：[TestMethod] 或 [DataTestMethod]（用于MSTest），[Fact] 或 [Theory]（用于Xunit）或 [Test]，[TestCase]， [TestCaseSource] 或 [Theory]（用于Nunit）。然而，无论是否含有测试属特性，非 public方法都不会被视为测试，也就不会被执行。async void 型的方法以及在签名里使用了泛型的方法也是如此.

不合规的代码示例

[TestMethod]
void TestNullArg()  // 不合规，此方法不是public型
{  /* …… */  }

[TestMethod]
public async void MyIgnoredTestMethod()  // 不合规，此方法是一个‘async void’方法
{ /* …… */ }

[TestMethod]
public void MyIgnoredGenericTestMethod<T>(T foo)  // 不合规，此方法的签名中有泛型
{ /* …… */ }

合规的解决方案

[TestMethod]
public void TestNullArg()
{  /* …… */  }

例外

忽略xUnit的 Fact 测试方法的可访问性，因为它们不需要是 public。

xUnit中的 [Theory] 测试方法以及nuint里的 [TestCase] 和 [TestCaseSource] 可以使用泛型。
