不必要的声明和初始化会使代码难以阅读，应将其删除。

具体来说，下列内容应予以省略：

  • 数组元素的类型
  • 数组大小
  • new DelegateType
  • new Nullable<Type>
  • 对象或集合的初始化器（initializer），即{}
  • lambda表达式参数的类型
  • 匿名方法的未使用参数的参数声明

不合规的代码示例

var l = new List<int>() {}; // 不合规，可删除{}
var o = new object() {}; // 不合规，可删除{}

var ints = new int[] {1, 2, 3}; // 不合规，可省略int
ints = new int[3] {1, 2, 3}; // 不合规，数组大小的声明可删除

int? i = new int?(5); // 不合规，new int?可以省略，它可以通过声明推断出来，有从T到T的隐式转换吗？
var j = new int?(5);

Func<int, int> f1 = (int i) => 1; //不合规，可简化

class Class
{
    private event EventHandler MyEvent;

    public Class()
    {
        MyEvent += new EventHandler((a,b)=>{ }); // 不合规，多余
    }
}

合规的解决方案

var l = new List<int>();
var o = new object();

var ints = new [] {1, 2, 3};
ints = new [] {1, 2, 3};

int? i = 5;
var j = new int?(5);

Func<int, int> f1 = (i) => 1;

class Class
{
    private event EventHandler MyEvent;

    public Class()
    {
        MyEvent += (a,b)=>{ };
    }
}
