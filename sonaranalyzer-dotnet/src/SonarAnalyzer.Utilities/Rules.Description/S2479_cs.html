未经编码的控制字符和空白字符往往会弄坏你的代码，它们不可见，或难以辨认，当字符串并非开发者预期时往往会引发bug。如果你确实需要它们，请使用它们的编码版本(例如：ASCII \n,\t,... 或 Unicode U+000D, U+0009,...).

当字符串包含以下字符时报告问题：

  • ASCII 控制字符 (字符索引 < 32 或 = 127)
  • Unicode 空白字符
  • Unicode C0 控制字符
  • Unicode 字符 U+200B, U+200C, U+200D, U+2060, U+FEFF, U+2028, U+2029

单独的空格字符，例如 Unicode U+0020, ASCII 32 不会报告问题。

不合规的代码示例

string tabInside = "A	B";  // 不合规，包含了制表符
string zeroWidthSpaceInside = "foo​bar"; // 不合规，使用了 U+200B 字符
Console.WriteLine(zeroWidthSpaceInside); // 打印 "foo?bar"

合规的解决方案

string tabInside = "A\tB";  // 合规，使用了转义值
string zeroWidthSpaceInside = "foo\u200Bbar";  // 合规，使用了转义值
Console.WriteLine(zeroWidthSpaceInside); // 打印 "foo?bar"

例外

逐字符字面量没有转义字符机制。
