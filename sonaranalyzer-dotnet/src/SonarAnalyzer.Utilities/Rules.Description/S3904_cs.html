如果不提供 AssemblyVersionAttribute，那么在每次建立特性化的程序集时都会使用默认的程序集版本。由于.NET Framework使用版本号来唯一地标识一个程序集，因此这可能会导致依赖关系被破坏（broken dependencies）。

不合规的代码示例

using System;
using System.Reflection;

[assembly: AssemblyTitle("MyAssembly")] // 不合规

namespace MyLibrary
{
}

合规的解决方案

using System;
using System.Reflection;

[assembly: AssemblyTitle("MyAssembly")]
[assembly: AssemblyVersionAttribute("1.2.125.0")]

namespace MyLibrary
{
}
