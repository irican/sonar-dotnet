readonly 字段只能在类的构造函数中赋值。如果某个类有一个只设置在构造函数里的字段而没有将它标记为 readonly，就可能会让人对该字段的用途产生困惑。为避免这种情况，这样的字段应该给予 readonly 标记明确地表明他们的用途，并且防止将来维护人员无意中将其更改。

不合规的代码示例

class Person
{
  int _birthYear;  // 不合规
  Person(int birthYear)
  {
    _birthYear = birthYear;
  }
}

合规的解决方案

class Person
{
  readonly int _birthYear;
  Person(int birthYear)
  {
    _birthYear = birthYear;
  }
}

例外

此规则忽略有特性的字段。
