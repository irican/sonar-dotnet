GC.SuppressFinalize 要求公共语言运行库（Common Language Runtime）不要调用对象的终结器（finalizer），如果某对象的终结已经由 IDisposable.Dispose 处理，那么在实现该对象的的dispose模式时，GC.SuppressFinalize是有用的。然而，如果对象的类型中没有定义终结器，那么它就没有效果，因此在这种情况下使用它只会让人困惑。

当对没有终结器的 sealed 对象调用 GC.SuppressFinalize 时，该规则会引发一个问题。

注意：＊＊ {rule:csharpsquid:S3971} 是此规则更严格化的版本。通常这二者中只触发一条就是有意义的。

不合规的代码示例

sealed class MyClass
{
  public void Method()
  {
    ...
    GC.SuppressFinalize(this); //不合规
  }
}

合规的解决方案

sealed class MyClass
{
  public void Method()
  {
    ...
  }
}
