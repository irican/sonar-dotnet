当一个 static 构造函数除了初始化 static 字段没有其他的用途时，它会带来不必要的开销。因为编译器会在每个 static 方法或实例的构造函数调用前进行一次检查。

因此，强烈建议使用内联初始化。

不合规的代码示例

namespace myLib
{
  public class Foo
  {
    static int i;
    static string s;

    static Foo() // 不合规
    {
      i = 3;
      ResourceManager sm =  new ResourceManager("strings", Assembly.GetExecutingAssembly());
      s = sm.GetString("mystring");
    }
  }
}

合规的解决方案

namespace myLib
{
  public class Foo
  {
    static int i =3;
    static string s = InitString();

    static string InitString()
    {
      ResourceManager sm = new ResourceManager("strings", Assembly.GetExecutingAssembly());
      return sm.GetString("mystring");
    }
  }
}
