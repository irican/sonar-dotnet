几乎没有必要调用 GC.Collect，相反调用它还可能会大大地影响应用程序的性能。因为它会产生一个阻塞性（blocking）的操作，即检查 内存中的每一个对象 以待清理它们。并且，这个阻塞性的清理操作在实际运行时你是无法控制的。

一般情况下，调用此方法的负面效果远超过它带来的好处。除非你在程序的运行中触发了一些特别的事件，而这些事件造成了许多原本生存时间很长的对象的死亡。

此规则会在调用了 GC.Collect 时引发一个问题。

不合规的代码示例

static void Main(string[] args)
{
  // ……
  GC.Collect(2, GCCollectionMode.Optimized); // 不合规
}
