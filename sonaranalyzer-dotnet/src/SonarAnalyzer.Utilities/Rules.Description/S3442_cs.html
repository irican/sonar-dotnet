由于 abstract 类无法实例化，所以他们没有必要拥有 public 或 internal 型的构造函数。在创建扩展类的实例时如果有应该运行的基本的初始化逻辑（basic initialization logic），你可以尽可能地将其放入一个构造函数中，但要将这个构造函数设为 private 或 protected 型。

不合规的代码示例

abstract class Base
{
    public Base() // 不合规，应设为private或protected型
    {
      //……
    }
}

合规的解决方案

abstract class Base
{
    protected Base()
    {
      //……
    }
}
