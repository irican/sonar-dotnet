永远不被执行或引用的private 或 internal 类型以及 private 成员称作死代码（dead code）：即应该删去的无用代码。清理这样的死代码能减小维护代码库（maintained codebase）的规模，有助于对程序的理解，还能阻止bug的引入。

不合规的代码示例

public class Foo
{
  private void UnusedPrivateMethod() {...} // 不合规

  private class UnusedClass {...} // 不合规
}

合规的解决方案

public class Foo
{
  public Foo()
  {
    UsedPrivateMethod();
  }

  private void UsedPrivateMethod()
  {
    var c = new UsedClass();
  }

  private class UsedClass {...}
}

例外

此规则在下列情况下不会引发问题：

  • 空构造函数
  • 有特性（Attribute）的成员
  • Main 方法
  • 在partial类的声明中，具有事件处理器（event handler）签名的方法，如 void Foo(object, EventArgs)。
  • 带有 System.SerializableAttribute 的类型的空序列化构造函数。
  • 程序集（assembly）中有 System.Runtime.CompilerServices.InternalsVisibleToAttribute 特性的内部成员。
