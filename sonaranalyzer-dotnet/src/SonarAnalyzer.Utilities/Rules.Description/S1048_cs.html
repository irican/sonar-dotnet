如果Finalize方法或Finalize的重写方法抛出一个异常，并且运行时（runtime）不由重写了默认策略的应用程序管理，那么运行时就会立刻终结进程而无法优雅地（graceful）完成清理（块和终结器不会执行）。如果终结器无法释放或销毁资源，那么此行为可确保进程的完整性。

此规则会报告终结器里的异常抛出语句。

不合规的代码示例

class MyClass
{
    ~MyClass()
    {
        throw new NotImplementedException(); // 不合规
    }
}

合规的解决方案

class MyClass
{
    ~MyClass()
    {
        // 无异常抛出
    }
}

请参阅

  • MISRA C++:2008, 15-5-1 - A class destructor shall not exit with an exception.
