不应对值为 null 的引用（reference）再进行访问（access）或解引用（dereference）操作。因为这样做会引发NullReferenceException 。最好的情况下，该异常会导致程序突然结束。最坏的情况下，它会泄露对攻击者有用的调试信息，或是允许攻击者绕过各种安全措施。

不合规的代码示例

object o = null;
if (condition)
{
  M1(o.ToString()); // 不合规，总为null
}
else
{
  o = new object();
}
M2(o.ToString());

例外

对扩展方法的调用不会报告该问题，因为他们在 null 值上仍能运行。

若要创建自定义的null值检查方法，请声明名为 ValidatedNotNullAttribute 的特性并用其在你的方法的声明中标记已验证为空的参数：

using System;

public sealed class ValidatedNotNullAttribute : Attribute { }

public static class Guard
{
    public static void NotNull<T>([ValidatedNotNull] this T value, string name) where T : class
    {
        if (value == null)
            throw new ArgumentNullException(name);
    }
}

public static class Utils
{
    public static string ToUpper(string value)
    {
        Guard.NotNull(value, nameof(value));
        if (value == null)
        {
            return value.ToString(); // 合规；此行代码不会执行
        }
        return value.ToUpper();
    }
}
