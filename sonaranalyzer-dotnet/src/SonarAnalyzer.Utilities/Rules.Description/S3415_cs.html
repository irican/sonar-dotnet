标准断言库方法（standard assertions library method），如 MSTest 和 NUnit 中的 AreEqual 和 AreSame，或是 XUnit 中的 Equal 和 Same 都希望第一个参数是期望值（expected value），第二个参数是实际值（actual value）。如果将二者调换，你的测试总是会产生一样的结果（成功或失败），但错误信息会让人困惑。

如果断言库方法第二个参数是一个硬编码值并且第一个参数不是的话，此规则会引发一个问题。

不合规的代码示例

Assert.AreEqual(runner.ExitCode, 0, "Unexpected exit code");  // 不合规，会产生类似这样的错误信息：Expected:<-1>. Actual:<0>.

合规的解决方案

Assert.AreEqual(0, runner.ExitCode, "Unexpected exit code");
