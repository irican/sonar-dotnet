在C#中，委托（delegate）可以加在一起链式地执行，也可以从链中减去某个委托表示不执行它。

从某条链中删去一串委托可能会导致如下所示的意外结果——而且很可能形成bug。

不合规的代码示例

MyDelegate first, second, third, fourth;
first = () => Console.Write("1");
second = () => Console.Write("2");
third = () => Console.Write("3");
fourth = () => Console.Write("4");

MyDelegate chain1234 = first + second + third + fourth; // 合规——链的顺序为“1234”
MyDelegate chain12 = chain1234 - third - fourth; // 合规——链的顺序为“12”


MyDelegate chain14 = first + fourth; // 新建一个MyDelegate实例instance which is a list under the covers
MyDelegate chain23 = chain1234 - chain14; // 不合规，first + fourth 在chain1234中不存在


// “chain23”的顺序是“1234”而不是“23”！
// 事实上，序列“1234”并不包含子序列“14”，因此减操作并没有成功
// （但注意序列“1234”有子序列“1”和“4”）
chain23 = chain1234 - (first + fourth); // 不合规

chain23(); // 会输出“1234”！

合规的解决方案

MyDelegate chain23 = chain1234 - first - fourth; // 合规——先删除了“1”，而后删除了“4”

chain23(); // 会输出“23”
