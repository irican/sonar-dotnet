没有用到的形参是有误导性的。无论传给形参的值如何，方法的行为应该都是一样的。

不合规的代码示例

void DoSomething(int a, int b) // "b" 没有用到
{
  Compute(a);
}

void DoSomething2(int a) // "a" 的值没有用到
{
  a = 10;
  Compute(a);
}

合规的解决方案

void DoSomething(int a)
{
  Compute(a);
}

void DoSomething2()
{
  var a = 10;
  Compute(a);
}

例外

virtual，override 方法以及接口的实现被此规则忽略。

override void DoSomething(int a, int b) // b未使用，但不会引发问题
{
  Compute(a);
}

而且扩展方法（extension methods）的 this 参数也被此规则忽略。

public static class Extensions
{
  public static void MyHelper(this HtmlHelper helper) //这里不会引发问题
  {
    // 这里没有使用helper
  }
}

带有特性（attribute）的方法不必考虑此规则。

public class MyDto
{
  public string Name { get; set; }

  [OnDeserialized]
  private void OnDeserialized(StreamingContext context)
  {
    // ...
  }
}

空方法或不受支持的方法也被此规则忽略。

public void DoSomething()
{}

public void Call()
{
  throw new NotImplementedException();
}

static void Main(string[] args) 方法显然也不会引发问题

请参阅

  • MISRA C++:2008, 0-1-11 - There shall be no unused parameters (named or unnamed) in nonvirtual functions.
  • MISRA C:2012, 2.7 - There should be no unused parameters in functions
  • CERT, MSC12-C. - Detect and remove code that has no effect or is never executed
