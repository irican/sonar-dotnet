使用自定义异常类型的目的就是要比标准类型传达更多的信息。但它必须是 public 型才能工作。

如果一个方法抛出了非public类型的异常，在调用者这一方所能做的最多就是 catch 该类最近的 public 基类（the closest public base of the class）。也就是说，你丢失了所有已创建异常类型要传递的自定义信息。

不合规的代码示例

internal class MyException : Exception   // 不合规
{
  // ……
}

合规的解决方案

public class MyException : Exception
{
  // ……
}

例外

所有不是直接地派生自 System.Exception，System.SystemException 或 System.ApplicationException 的异常类型被此规则忽略。
