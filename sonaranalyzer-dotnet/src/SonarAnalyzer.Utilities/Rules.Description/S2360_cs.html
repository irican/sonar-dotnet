应使用重载机制来代替可选参数的使用，这样做有以下几个原因：

  • 可选参数的值会被加入到方法调用处的代码中，因此，如果更改了默认值，所有的引用程序集（referencing assemblies）都需要重建，否则会使用最初的原始值。
  • 公共语言规范（CLS）允许编译器忽视默认的参数值，因此它也要求调用者显式地指定这些值。例如，如果你想从另一种.NET兼容的语言（如C++/CLI）中使用带默认参数的方法，你就需要提供所有的参数。在使用方法重载时，你可以实现与默认参数类似的性质。
  • 可选参数会防止混淆函数约定的定义。举个简单的例子：如果有两个可选参数，其中一个被定义了，那么另一个是可选的还是必需的？

不合规的代码示例

void Notify(string company, string office = "QJZ") // 不合规
{
}

合规的解决方案

void Notify(string company)
{
  Notify(company, "QJZ");
}
void Notify(string company, string office)
{
}

例外

此规则忽略外部不可见的方法。
