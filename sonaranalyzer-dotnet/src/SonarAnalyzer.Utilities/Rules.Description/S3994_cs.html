字符串表示的URI或URL易出现解析和编码错误，这些错误可能会导致漏洞的产生。System.Uri 类是一种安全的替代方案，应予以优先考虑。若有某个类包含了明显以 string 形式传递Uri的方法，则在每一个这样的类中，应至少提供一个使用 System.Uri 参数的方法重载。

当某个方法有名称包含“uri”，“Uri”，“urn”，“Urn”，“url”或“Url”的字符串参数，并且该类型没有声明相应的重载，而是采用System.Uri 参数时，此规则会引发一个问题。

不合规的代码示例

using System;

namespace MyLibrary
{
   public class MyClass
   {

      public void FetchResource(string uriString) { } // 不合规
   }
}

合规的解决方案

using System;

namespace MyLibrary
{
   public class MyClass
   {

      public void FetchResource(string uriString)
      {
          FetchResource(new Uri(uriString));
      }

      public void FetchResource(Uri uri) { }
   }
}
