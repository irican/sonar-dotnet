没有任何测试方法的测试类是没有意义的。它可能使得维护人员认为某个类被测试覆盖。

支持的测试框架有 NUnit 和 MSTest（不适用于 xUnit）。

以下情况不会引发问题：

  • 对于 NUnit，标记有 TestFixture 但不含有任何标有 Test，TestCase， TestCaseSource 和 Theory 方法的类不会引发问题。
  • 对于 MSTest，标记有 TestClass 但不含有任何标有 TestMethod 和 DataTestMethod 方法的类不会引发问题。

不合规的代码示例

[TestFixture]
public class SomeClassTest { } // 不合规-没有测试的方法

[TestClass]
public class SomeOtherClassTest { } // 不合规-没有测试的方法

合规的解决方案

[TestFixture]
public class SomeClassTest
{
    [Test]
    public void SomeMethodShouldReturnTrue() { }
}

[TestClass]
public class SomeOtherClassTest
{
    [TestMethod]
    public void SomeMethodShouldReturnTrue() { }
}

例外

  • 抽象类
  • 继承自某个没有测试方法的类的派生类
  • MSTest中，那些包含被 AssemblyInitialize 或 AssemblyCleanup 标记的方法的类。
