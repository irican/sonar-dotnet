在两个不相关的类中使用相同的变量名没有问题，但如果在处于同一继承体系的类中这样做，你可能就会感到困惑和混乱。

不合规的代码示例

public class Fruit
{
  protected Season ripe;
  protected Color flesh;

  // ……
}

public class Raspberry : Fruit
{
  private bool ripe; // 不合规
  private static Color FLESH; // 不合规
}

合规的解决方案

public class Fruit
{
  protected Season ripe;
  protected Color flesh;

  // ……
}

public class Raspberry : Fruit
{
  private bool ripened;
  private static Color FLESH_COLOR;
}

例外

此规则忽略在父类和子类中同为 static 型的同名字段。它还忽略父类中的 private 型字段，除此以外， 子类中的字段都应重新命名。

public class Fruit
{
  private Season ripe;
  // ...
}

public class Raspberry : Fruit
{
  private Season ripe;  // 父字段的‘ripe’对Raspberry始终不可见时合规
  // ...
}
