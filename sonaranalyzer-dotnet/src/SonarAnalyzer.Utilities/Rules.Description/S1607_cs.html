当测试因为基础结构或其他方面的问题失败时，你可能想暂时先忽略它。但如果没有某种记号说明该测试被忽略的原因，它可能不会再被激活。如果没有对项目的全面了解，这些测试很难去处理，并且最终还会污染他们的项目。

每有一个被忽略的测试此规则都会引发问题，如没有 WorkItem 特性，或是在 Ignore 特性右侧缺少注释说明跳过此处的原因。

不合规的代码示例

[TestMethod]
[Ignore]  // 不合规
public void Test_DoTheThing()
{
  // ……
}

合规的解决方案

[TestMethod]
[Ignore]  // 修复TCKT-1234后可重新启用
public void Test_DoTheThing()
{
  // ……
}

或者

[TestMethod]
[Ignore]
[WorkItem(1234)]
public void Test_DoTheThing()
{
  // ……
}

例外

此规则在以下情况下不会引发问题：

- 测试的方法也标有 WorkItem 特性

- 在 Ignore 特性的右侧有注释说明
