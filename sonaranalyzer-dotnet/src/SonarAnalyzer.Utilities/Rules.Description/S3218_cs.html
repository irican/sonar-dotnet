可以将内部类（inner class）的成员命名为与其外部类的 static 成员相同的名称——虽然可以，但这是一个坏主意。因为维护人员可能会困惑于某处使用的是哪一个成员。因此相反地，应重命名内部类的成员，并更改所有对它的引用。

不合规的代码示例

class Outer
{
  public static int A;

  public class Inner
  {
    public int A; //不合规
    public int MyProp
    {
      get { return A; }  // 会返回内部类的A。这是编写者的本意吗？
    }
  }
}

重命名后

class Outer
{
  public static int A;

  public class Inner
  {
    public int B;
    public int MyProp
    {
      get { return A; }  // 仍然能编译运行但功能已经不同了
    }
  }
}

合规的解决方案

class Outer
{
  public static int A;

  public class Inner
  {
    public int InnerA;
    public int MyProp
    {
      get { return InnerA; }
    }
  }
}
