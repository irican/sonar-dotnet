派生类被用作参数而不是另一个基类实际上是限制了方法的用途。如果不需要派生类中新添加的功能，那么刚才所说的限制也是不需要的，应该将其移除。

当某方法的声明里含有派生类作为参数，并且只访问它基类的成员时，此规则会引发一个问题。

不合规的代码示例

using System;
using System.IO;

namespace MyLibrary
{
  public class Foo
  {
    public void ReadStream(FileStream stream) // 不合规：仅使用了方法System.IO.Stream
    {
      int a;
      while ((a = stream.ReadByte()) != -1)
      {
            // 此处进行某些操作
      }
    }
  }
}

合规的解决方案

using System;
using System.IO;

namespace MyLibrary
{
  public class Foo
  {
    public void ReadStream(Stream stream)
    {
      int a;
      while ((a = stream.ReadByte()) != -1)
      {
            // 此处进行某些操作
      }
    }
  }
}
