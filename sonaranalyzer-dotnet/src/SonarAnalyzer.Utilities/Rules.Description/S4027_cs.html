异常类型应该提供以下构造函数：

  • public MyException()
  • public MyException(string)
  • public MyException(string, Exception)
  • protected 或 private MyException(SerializationInfo, StreamingContext)

第四个构造函数在未密封类中应该是protected，在密封类中应该是private。

如果没有完整的构造函数集，就很难处理异常。

不合规的代码示例

using System;

namespace MyLibrary
{
  public class MyException // 不合规：缺少几个构造函数
  {
    public MyException()
    {
    }
  }
}

合规的解决方案

using System;
using System.Runtime.Serialization;

namespace MyLibrary
{
  public class MyException : Exception
  {
      public MyException()
      {
      }

      public MyException(string message)
          :base(message)
      {
      }

      public MyException(string message, Exception innerException)
          : base(message, innerException)
      {
      }

      protected MyException(SerializationInfo info, StreamingContext context)
          : base(info, context)
      {
      }
  }
}
