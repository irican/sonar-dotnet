捕获 System.Exception 似乎是个能处理多个可能异常的有效方法。然而不幸的是，它会捕获所有的异常类型，甚至包括那些你并不打算捕获的。为避免误解应使用异常过滤，或者把每种异常的类型写在不同的 catch 语句块中。

不合规的代码示例

try
{
  // 此处是一些可能抛出FileNotFoundException或IOException的操作
}
catch (Exception e) // 不合规
{
  // 记录异常……
}

合规的解决方案

try
{
  // 此处进行某些操作
}
catch (Exception e) when (e is FileNotFoundException || e is IOException)
{
  // 此处进行某些操作
}

例外

还可以在 catch 语句块的最后一条语句里捕获 System.Exception 并 throw 它。但这是最差的选项，因为这是一种老式的代码，比起异常过滤它有额外的性能损耗。

try
{
  // 此处进行某些操作
}
catch (Exception e)
{
  if (e is FileNotFoundException || e is IOException)
  {
    // 此处进行某些操作
  }
  else
  {
    throw;
  }
}
