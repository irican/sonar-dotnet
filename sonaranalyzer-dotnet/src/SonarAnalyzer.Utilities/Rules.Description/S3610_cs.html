对可为null的对象调用 GetType() 会返回基础的值类型（underlying value type）。因此，将返回的 Type 对象与 typeof(Nullable<SomeType>) 比较没有意义。这次比较要么会引发异常，要么其结果在编译时就已知晓。

不合规的代码示例

int? nullable = 42;
bool comparison = nullable.GetType() == typeof(Nullable<int>); // 不合规，总为false
comparison = nullable.GetType() != typeof(Nullable<int>); // 不合规，总为true

nullable = null;
comparison = nullable.GetType() != typeof(Nullable<int>); // 不合规，对null调用GetType总是会引发一个异常
