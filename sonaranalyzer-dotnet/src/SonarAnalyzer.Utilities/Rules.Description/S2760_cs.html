同一个条件连续地检查两次会很让人困惑——为什么要分开检查呢？——也有可能是个错误，比如第二次本应检查其他的条件。

不合规的代码示例

if (a == b)
{
  doTheThing(b);
}
if (a == b) // 不合规；这是开发者的本意吗？
{
  doTheThing(c);
}

合规的解决方案

if (a == b)
{
  doTheThing(b);
  doTheThing(c);
}

或者

if (a == b)
{
  doTheThing(b);
}
if (b == c)
{
  doTheThing(c);
}

例外

把检查变量的值作为条件是一种常见的模式，因此检查失败后对变量重赋值，而后再检查的情况可以忽略。
