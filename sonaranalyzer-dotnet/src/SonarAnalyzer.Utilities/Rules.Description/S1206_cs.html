Equals(object) 和 GetHashCode() 有个使用约定： 如果调用 Equals(object) 方法后得出两个对象相等，那么对它们分别调用 GetHashCode() 方法的结果也必须是一样的。如果不一样，就会有许多的集合（collection）就无法正确地处理类的实例。

要遵守此约定，Equals(object) 和 GetHashCode() 方法应该一起继承或重写。

不合规的代码示例

class MyClass {    // 不合规 - "hashCode()"也要重写

  @Override
  public boolean equals(Object obj) {
    /* …… */
  }

}

合规的解决方案

class MyClass {    // 合规

  @Override
  public boolean equals(Object obj) {
    /* …… */
  }

  @Override
  public int hashCode() {
    /* …… */
  }

}

请参阅

  • MITRE, CWE-581 - Object Model Violation: Just One of Equals and Hashcode Defined
  • CERT, MET09-J. - Classes that define an equals() method must also define a hashCode() method
