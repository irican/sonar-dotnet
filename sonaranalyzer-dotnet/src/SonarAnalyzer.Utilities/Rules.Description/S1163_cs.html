从finally语句块内抛出的异常会覆盖之前在 try 或 catch 语句块中抛出的异常，并且，被覆盖异常的信息和堆栈跟踪内容（stack trace）也会丢失。

不合规的代码示例

try
{
  /* 此处是一些在结束时抛出异常的操作 */
  throw new ArgumentException();
}
finally
{
  /* 清理 */
  throw new InvalidOperationException();       // 不合规，此处会覆盖ArgumentException
}

合规的代码示例

try
{
  /* 此处是一些在结束时抛出异常的操作 */
  throw new ArgumentException();
}
finally
{
  /* 清理 */                       // 合规
}

请参阅

  • CERT, ERR05-J. - Do not let checked exceptions escape from a finally block
