根据Robert C. Martin在他的著作“Principles of Object Oriented Design”中介绍的单一功能原则（the Single Responsibility Principle），一个类应该只承担一项功能（responsibility）：

如果一个类承担了多个功能，这些功能间就会产生连结（couple）。

对某项功能的的改变可能会削弱或抑制该类与其他类交互（meet the others）的能力。

这种连结会产生脆弱的设计，改变它时它会以意外的方式崩溃。

依赖于许多其他类的类往往集中了太多的职责，应该将其分成几个较小的类。

嵌套类所依赖的类不会被视为它的外部类的依赖类。

不合规的代码示例

所依赖类的阈值为5时：

public class Foo    // 不合规 - Foo依赖的类太多，如： T1, T2, T3, T4, T5, T6 和 T7
{
  private T1 a1;    // Foo与T1有关
  private T2 a2;    // Foo与T2有关 
  private T3 a3;    // Foo与T3有关 

  public T4 Compute(T5 a, T6 b)    // Foo与T4，T5和T6都有关联
  {
    T7 result = a.Process(b);    // Foo与T7有关
    return result;
  }

  public static class Bar    // 合规 - Bar只依赖两个类：T8 和 T9
  {
    public T8 a8;
    public T9 a9;
  }
}
