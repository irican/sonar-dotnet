调试语句在开发时是很有用的。但在产品代码中仍保留他们——尤其是客户端的代码——则可能会暴露敏感的信息。

不合规的代码示例

private void DoSomething()
{
    // ……
    Console.WriteLine("so far, so good..."); // 不合规
    // ……
}

例外

此规则忽略以下情况：

  • 控制台应用程序
  • 标有 [Conditional ("DEBUG")] 的方法里的调用
  • DEBUG预处理分支（#if DEBUG）里的调用
