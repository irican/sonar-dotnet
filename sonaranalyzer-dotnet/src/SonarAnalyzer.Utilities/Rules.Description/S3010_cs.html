给构造函数中的 static 字段赋值可能会导致在运行时出现不可靠的行为，因为这样将更改类的所有实例的值。

因此应删除字段的 static 修饰符，或者静态地初始化它。

不合规的代码示例

public class Person
{
  private static DateTime dateOfBirth;
  private static int expectedFingers;

  public Person(DateTime birthday)
  {
    dateOfBirth = birthday;  // 不合规，这样每个对象都有相同的birthday
    expectedFingers = 10;  // 不合规
  }
}

合规的解决方案

public class Person
{
  private DateTime dateOfBirth;
  private static int expectedFingers = 10;

  public Person(DateTime birthday)
  {
    this.dateOfBirth = birthday;
  }
}
