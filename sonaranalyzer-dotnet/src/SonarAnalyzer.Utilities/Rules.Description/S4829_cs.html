读取标准输入是安全敏感的。它在过去导致了以下漏洞：

  • CVE-2005-2337
  • CVE-2017-11449

攻击者通常会精心设计输入，使其能够利用软件漏洞。因此，从标准输入(stdin)读取的任何数据都可能是危险的，应该进行验证。

此规则标记从标准输入中读取的代码。

检查是否有下列问题：

  • 从标准输入中读取的数据在使用之前没有经过净化。

如果对其中任何一个问题的回答是肯定的，那么就有风险。

推荐的安全编码实践

在使用标准输入之前对所有读取的数据进行净化。

可疑的代码示例

using System;
public class C
{
    public void Main()
    {
        Console.In; // 可疑的
        var code = Console.Read(); // 可疑的
        var keyInfo = Console.ReadKey(...); // 可疑的
        var text = Console.ReadLine(); // 可疑的
        Console.OpenStandardInput(...); // 可疑的
    }
}

例外

当忽略 Console.Read Console.ReadKey 或 Console.ReadLine 方法的返回值时，此规则不会引发问题。

using System;
public class C
{
    public void Main()
    {
        Console.ReadKey(...); // 返回值被忽略
        Console.ReadLine(); // 返回值被忽略
    }
}
