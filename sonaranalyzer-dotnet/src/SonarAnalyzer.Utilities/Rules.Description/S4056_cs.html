当不提供System.Globalization.CultureInfo或IFormatProvider对象时，重载成员提供的默认值可能不会在所有地区产生希望的效果。

应按照以下指导原则提供区域性特定的信息：

  • 如果该值将显示给用户，则使用当前区域性。请参阅CultureInfo.CurrentCulture。
  • 如果该值将由软件存储和访问(持久化到文件或数据库)，则使用不变区域性。请参阅CultureInfo.InvariantCulture。
  • 如果不知道值的目的地，请让数据使用者或提供者指定区域性。

当方法或构造函数调用一个或多个具有重载的成员，这些成员接受System.IFormatProvider 参数，而方法或构造函数不调用接受IFormatProvider参数的重载时，该规则会引发一个问题。该规则忽略.NET Framework方法的调用，这些调用被记录为忽略IFormatProvider参数，以及下列方法：

  • Activator.CreateInstance
  • ResourceManager.GetObject
  • ResourceManager.GetString

不合规的代码示例

using System;

namespace MyLibrary
{
    public class Foo
    {
        public void Bar(String string1)
        {
            if(string.Compare(string1, string2, false) == 0) // 不合规
            {
                Console.WriteLine(string3.ToLower()); // 不合规
            }
        }
    }
}

合规的解决方案

using System;
using System.Globalization;

namespace MyLibrary
{
    public class Foo
    {
        public void Bar(String string1, String string2, String string3)
        {
            if(string.Compare(string1, string2, false,
                              CultureInfo.InvariantCulture) == 0)
            {
                Console.WriteLine(string3.ToLower(CultureInfo.CurrentCulture));
            }
        }
    }
}

例外

当重载被标记为过时时，此规则不会引发问题。
