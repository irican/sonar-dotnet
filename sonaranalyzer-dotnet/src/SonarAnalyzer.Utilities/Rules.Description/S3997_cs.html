URI或URL的字符串表示容易出现解析和编码错误，这些错误可能导致漏洞的产生。System.Uri 类是一种安全的替代方案，应予以优先考虑。

当两个重载仅有字符串／Uri 参数不同并且字符串重载没有调用 Uri 重载时，此规则会引发一个问题。一般认为除参数类型不同外，用字符串参数表示URI具有准确的匹配性。但显然应该使用更安全的重载。

不合规的代码示例

using System;

namespace MyLibrary
{
   public class MyClass
   {
      public void FetchResource(string uriString) // 不合规
      {
         // 没有调用FetResource(Uri)
      }

      public void FetchResource(Uri uri) { }
   }
}

合规的解决方案

using System;

namespace MyLibrary
{
   public class MyClass
   {
      public void FetchResource(string uriString)
      {
          FetchResource(new Uri(uriString));
      }

      public void FetchResource(Uri uri) { }
   }
}
