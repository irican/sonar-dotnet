有位域标签的枚举类不应依赖语言来初始化成员的值。隐式的初始化会给第一个成员赋值为0，然后递增并依次为之后的成员赋值，这种隐式的行为不允许以有效的方式使用按位或运算符来组合枚举成员。

相反地，0和2的幂（例如 1，2，4，8，16，……）应显式地初始化所有成员。

不合规的代码示例

[Flags]
enum FruitType    // 不合规
{
  None,
  Banana,
  Orange,
  Strawberry
}
class Program
{
    static void Main()
    {
        var bananaAndStrawberry = FruitType.Banana | FruitType.Strawberry;
        // 只会显示 Strawberry！
        Console.WriteLine(bananaAndStrawberry.ToString());
    }
}

合规的解决方案

[Flags]
enum FruitType
{
  None = 0,
  Banana = 1,
  Orange = 2,
  Strawberry = 4
}
class Program
{
    static void Main()
    {
        var bananaAndStrawberry = FruitType.Banana | FruitType.Strawberry;
        // 会显示Banana和Strawberry，同预想的一致
        Console.WriteLine(bananaAndStrawberry.ToString());
    }
}

例外

默认的初始化会在前三个值中用0，1，2，3，4……来匹配0，1，2，4，8……，因此如果枚举的前三个成员没有初始化则不会报告任何问题。
