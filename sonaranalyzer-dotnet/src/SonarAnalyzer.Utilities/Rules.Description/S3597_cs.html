ServiceContract 特性指明由一个类或接口来定义一个Windows通讯接口（Windows Communication Foundation，即WCF）服务的通信约定。该类或该接口的服务操作由添加给方法的 OperationContract 特性来定义。定义没有任何服务操作的通讯约定是没有意义的，因此，在一个有 ServiceContract 的类或接口中应至少有一个方法标记有 OperationContract。类似地，WCF只服务于定义在 ServiceContract 类或接口中定义的 OperationContract 方法。因此此规则还会检查 ServiceContract 是否添加到了包含 OperationContract 方法的类型中。

不合规的代码示例

[ServiceContract]
interface IMyService // 不合规
{
  int MyServiceMethod();
}

合规的解决方案

[ServiceContract]
interface IMyService
{
  [OperationContract]
  int MyServiceMethod();
}
