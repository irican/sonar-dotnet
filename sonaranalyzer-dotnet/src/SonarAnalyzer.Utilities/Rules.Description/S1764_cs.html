在二元运算符的两侧使用同样的值几乎都是错的。如果是逻辑运算符，这种情况要么是一个复制／粘贴错误——或者说是因此产生的bug，要么就是应该简化的无用代码。如果是位运算符和大多数的二元算术运算符（bitwise operators and most binary mathematical），运算符两侧值相同时运算结果是可以预料到的，所以应该将其简化。

不合规的代码示例

if ( a == a ) // 总是为真
{
  doZ();
}
if ( a != a ) // 总是为假
{
  doY();
}
if ( a == b && a == b ) // 如果第一个表达式为真，则第二个也是真
{
  doX();
}
if ( a == b || a == b ) // 如果第一个表达式为真，则第二个也是真
{
  doW();
}

int j = 5 / 5; //总是为 1
int k = 5 - 5; //总是为 0

c.Equals(c);    //总是为真
Object.Equals(c, c); //总是为真

例外

此规则忽略这些运算符： *， +， =， << 和 >>
