由于编译器会自动调用基类的无参数的构造函数，因此没有必要显式地指明对它的调用。并且，如果某个类中只定义一个 public 型的无参数的构造函数，那么它将会被移除，因为编译器可以自动生成这样的构造函数。类似地，空的 static 构造函数和空的析构函数也是无用且多余的。

不合规的代码示例

class X
{
  public X() { } // 不合规
  static X() { }  // 不合规
  ~X() { } // 不合规

  ...
}

class Y : X
{
  public Y(int parameter) : base() // 不合规
  {
    /* 用参数parameter进行一些操作 */
  }
}

合规的解决方案

class X
{
  ...
}

class Y : X
{
  public Y(int parameter)
  {
    /* 用参数parameter进行一些操作 */
  }
}
