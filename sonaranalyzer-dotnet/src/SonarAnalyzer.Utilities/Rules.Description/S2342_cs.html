共享命名约定允许团队高效地协作。此规则检查所有的 enum 类名称是否与提供的正则表达式（regular expression）匹配。

将Microsoft推荐的配置作为默认配置：

  • 帕斯卡命名法（Pascal casing），即以大写字符开头，例如 BackColor
  • 两个字母构成的缩写可以将它们全都大写，例如 GetID
  • 较长的缩写需要将其小写，例如 GetHtml
  • 如果枚举类被标识为[Flags]那么它的名称应该写为复数（例如 MyOptions），否则将其写为单数（例如 MyOption）

不合规的代码示例

对无标识的枚举类使用的正则表达式： ^([A-Z]{1,3}[a-z0-9]+)*([A-Z]{2})?$

public enum foo // 不合规
{
    FooValue = 0
}

对有[Flags]标识的枚举类使用的正则表达式： ^([A-Z]{1,3}[a-z0-9]+)*([A-Z]{2})?s$

[Flags]
public enum Option // 不合规
{
    None = 0,
    Option1 = 1,
    Option2 = 2
}

合规的解决方案

public enum Foo
{
    FooValue = 0
}

[Flags]
public enum Options
{
    None = 0,
    Option1 = 1,
    Option2 = 2
}
