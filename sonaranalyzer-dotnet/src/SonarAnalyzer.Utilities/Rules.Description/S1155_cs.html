可以用 .Count() 来检查集合是否为空，但用 .Any() 意思更明显，代码也更好理解。但是仍有些情况需要特别注意：

- 如果集合是一个 EntityFramework 或其他的ORM查询（ORM query），调用 .Count() 可能会执行大量的SQL查询，在应用程序数据库（application database）上产生大量的花费。调用 .Any() 也会连接到数据库，但产生的SQL执行更有效率。

- 如果集合是包含了创建对象的 .Select() 语句的LINQ查询的一部分，就会有大量的内存被毫无必要地分配。调用.Any() 会更有效率，因为这样会执行更少的可枚举类型的遍历（iterations of the enumerable）。

不合规的代码示例

private static bool HasContent(IEnumerable<string> strings)
{
  return strings.Count() > 0;  // 不合规
}

private static bool HasContent2(IEnumerable<string> strings)
{
  return strings.Count() >= 1;  // 不合规
}

private static bool IsEmpty(IEnumerable<string> strings)
{
  return strings.Count() == 0;  // 不合规
}

合规的解决方案

private static bool HasContent(IEnumerable<string> strings)
{
  return strings.Any();
}

private static bool IsEmpty(IEnumerable<string> strings)
{
  return !strings.Any();
}
