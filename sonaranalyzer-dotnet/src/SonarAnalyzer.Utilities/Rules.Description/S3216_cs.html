在一个有 await 关键字的 Task 执行之后，你可以在原始的、调用的线程或者其他的任意线程里继续执行。除非其余的代码需要这个 Task 被引发的环境，否则就应该在 Task 线程里使用 Task.ConfigureAwait(false) 保持执行，以避免切换上下文切换和死锁。

当类库（class library）中的代码 await 了一个 Task 并且在原始的调用线程里继续执行时，此规则会引发一个问题。

不合规的代码示例

var response = await httpClient.GetAsync(url);  // 不合规

合规的解决方案

var response = await httpClient.GetAsync(url).ConfigureAwait(false);
