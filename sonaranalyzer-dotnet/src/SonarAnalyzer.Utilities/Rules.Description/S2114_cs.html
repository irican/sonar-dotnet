将集合作为参数传递给集合自己的方法是一种错误（本意可能是传递其他类型的参数），或者只是一个无意义的代码。

另外，由于一些方法要求参数需要在执行过程中保持不变，因此将集合作为参数传给它自己可能会导致以外的行为。

不合规的代码实例

var list = new List<int>();

list.AddRange(list); // 不合规
list.Concat(list); // 不合规

list.Union(list); // 不合规；总是返回 list
list.Except(list); // 不合规；总为空
list.Intersect(list); // 不合规；总是 list
list.SequenceEqual(list); // 不合规；总为真

var set = new HashSet<int>();
set.UnionWith(set); // 不合规；没有变化
set.ExceptWith(set); // 不合规；总为空
set.IntersectWith(set); // 不合规；没有变化
set.IsProperSubsetOf(set); // 不合规；总为假
set.IsProperSupersetOf(set); // 不合规；总为假
set.IsSubsetOf(set); // 不合规；总为真
set.IsSupersetOf(set); // 不合规；总为真
set.Overlaps(set); // 不合规；总为真
set.SetEquals(set); // 不合规；总为真
set.SymmetricExceptWith(set); // 不合规；总为空
