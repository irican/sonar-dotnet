NullReferenceException 应避免而不是捕获它。任何显式地捕获到 NullReferenceException 的情况都能很轻易地变为对 null 值的检测，并且在catch块中的任何行为也都可以很轻易地转到条件的 "is null" 分支下。

不合规的代码示例

public int GetLengthPlusTwo(string str)
{
    int length = 2;
    try
    {
       length += str.Length;
    }
    catch (NullReferenceException e)
    {
        log.info("argument was null");
    }
    return length;
}

合规的解决方案

public int GetLengthPlusTwo(string str)
{
    int length = 2;

    if (str != null)
    {
        length += str.Length;
    }
    else
    {
        log.info("argument was null");
    }
    return length;
}
