遵循标准的命名约定不仅让代码更易懂，还便于人们的使用。例如，class FirstAttribute : Attribute 可以仅和 First 一起使用，但你必须使用 class AttributeOne : Attribute 的完整名称。

当类扩展 Attribute， EventArgs 或 Exception 没有以他们的父类名称结尾时，此规则会引发一个问题。

不合规的代码示例

class AttributeOne : Attribute  // 不合规
{
}

合规的解决方案

class FirstAttribute : Attribute
{
}

例外

如果某个类的直接基类不遵循此约定，那么无论该类是否符合此约定都不会报告该类的问题。

class Timeout : Exception // 不合规
{
}
class ExtendedTimeout : Timeout // 忽略，不符合约定，但直接基类也没有遵循该约定
{
}
