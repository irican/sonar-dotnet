将一个初始化后不再变更的变量标记为 const 是在告诉之后的维护人员“不，该变量不能更新，也不应该被更新”。在这种情况下为了保证代码的明确易懂，应该使用关键字 const。

不合规的代码示例

public bool Seek(int[] input)
{
  int target = 32;  // 不合规
  foreach (int i in input)
  {
    if (i == target)
    {
      return true;
    }
  }
  return false;
}

合规的解决方案

public bool Seek(int[] input)
{
  const int target = 32;
  foreach (int i in input)
  {
    if (i == target)
    {
      return true;
    }
  }
  return false;
}
