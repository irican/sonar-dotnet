调用者信息属性：CallerFilePathAttribute 和 CallerLineNumberAttribute 提供了一种通过可选参数来获取方法调用者相关信息的方式。但就可选参数而言，它们只有在调用时没有被显式定义的情况下才会生成。因此，指定参数值会让这些属性无法运作。

不合规的代码示例

void TraceMessage(string message,
  [CallerFilePath] string filePath = null,
  [CallerLineNumber] int lineNumber = 0)
{
  /* …… */
}

void MyMethod()
{
  TraceMessage("my message", "A.B.C.Foo.cs", 42); // 不合规
}

合规的解决方案

void TraceMessage(string message,
  [CallerFilePath] string filePath = "",
  [CallerLineNumber] int lineNumber = 0)
{
  /* …… */
}

void MyMethod()
{
  TraceMessage("my message");
}

例外

为避免检测WPF/UWP应用程序时的误报，CallerMemberName 不予检查。
