工具类（Utility classes），即 static 型成员的集合类，是不应被实例化的。

C#为每个没有显式定义构造函数的类都添加了一个隐式的、公有的构造函数。因此，如果你想派生这种工具类的子类，你应该为这个工具类定义至少一个 protected 型的构造函数。否则就应在类的声明处加上 static 关键字使其无法被实例化。

不合规的代码示例

public class StringUtils // 不合规
{
  public static string Concatenate(string s1, string s2)
  {
    return s1 + s2;
  }
}

合规的解决方案

public static class StringUtils
{
  public static string Concatenate(string s1, string s2)
  {
    return s1 + s2;
  }
}

或者

public class StringUtils
{
  protected StringUtils()
  {
  }
  public static string Concatenate(string s1, string s2)
  {
    return s1 + s2;
  }
}
