由于参数名可以在重构过程中更改，因此不应该用字符串逐字符的拼写。相反地，如果使用nameof()，输出的字符串将始终正确。

当 throw 语句中的任一字符串与方法的某个参数的名称完全匹配时，此规则会引发一个问题。

不合规的代码示例

void DoSomething(int someParameter)
{
    if (someParameter < 0)
    {
        throw new ArgumentException("Bad argument", "someParameter");  // 不合规
    }
	if (anotherParam == null)
    {
        throw new Exception("anotherParam should not be null"); // 不合规
    }
}

合规的解决方案

void DoSomething(int someParameter)
{
    if (someParameter < 0)
    {
        throw new ArgumentException("Bad argument", nameof(someParameter));
    }
	if (anotherParam == null)
    {
        throw new Exception($"{nameof(anotherParam)} should not be null");
    }
}

例外

用户的C#版本低于6.0时忽略此问题。

当参数的名字包含在throw语句时，这条规则仅在参数至少为5个字符长的时候报告问题，以避免误报。
