为保证代码的可用性，含泛型参数的接口和委托应尽可能使用 out 和 in 修饰符，以使得接口和委托协变（covariant）和逆变（contravariant）。

当类型参数仅用作接口和委托中的返回类型时，可使用关键字 out。这样做能使得参数协变，并允许以子类创建的接口和委托的实例当作以基类创建的实例来使用。最值得注意的例子是 IEnumerable<out T>，它允许用 IEnumerable<string> 的实例给 IEnumerable<object> 变量赋值。

当类型参数仅用作接口中的方法参数和委托的参数时，可使用关键字 in。这样做能使得参数逆变，并允许以基类创建的接口和委托的实例当作以子类创建的实例使用。即协变的反转。最值得注意的例子是委托 Action<in T>，它允许 Action<object> 的实例对 Action<string> 变量赋值。

不合规的代码示例

interface IConsumer<T>  // 不合规
{
    bool Eat(T fruit);
}

合规的解决方案

interface IConsumer<in T>
{
    bool Eat(T fruit);
}
