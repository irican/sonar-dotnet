可以对只有一行的块语句省略花括号，例如只有一条语句的 if 语句或 for 循环，但这么做可能会让人误解并产生bug。

对于只有一行的块语句，当它之后数行的行首空格表示编程者有意将这些行也包括在块里时，由于花括号已省略，这些行的语句只会无条件地执行。这种现象发生时此规则会引发一个问题。

不合规的代码示例

if (condition)
  FirstActionInBlock();
  SecondAction();  // 不合规；该语句会无条件地执行
ThirdAction();

if(condition) FirstActionInBlock(); SecondAction();  // 不合规；secondAction会无条件地执行

if(condition) FirstActionInBlock();  // 不合规
  SecondAction();  // 无条件执行

string str = null;
for (int i = 0; i < array.Length; i++)
  str = array[i];
  DoTheThing(str);  // 不合规；该语句只会对数组的最后一个元素执行操作

合规的解决方案

if (condition)
{
  FirstActionInBlock();
  SecondAction();
}
ThirdAction();

string str = null;
for (int i = 0; i < array.Length; i++)
{
  str = array[i];
  DoTheThing(str);
}
