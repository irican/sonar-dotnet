共享命名约定允许团队高效地协作。此规则检查方法和属性名是否用帕斯卡命名法（PascalCased）命名。为了避免误解，两个连续的大写字符只要不是名称的全部，那么就是合规的。因此 MyXMethod是合规的，但 XM 就不合规。

不合规的代码示例

public int doSomething(){...}

合规的解决方案

public int DoSomething(){...}

例外

此规则忽略用 ComImportAttribute 或 InterfaceTypeAttribute标记的类型的成员。extern方法同样不予检查。此外，当 '_' 字符出现在名称中时，驼峰式命名法（camel casing）不再强制使用。

void My_method(){...} // 有效
void My_method_(){...} // 无效，名称首尾的下划线不合规
