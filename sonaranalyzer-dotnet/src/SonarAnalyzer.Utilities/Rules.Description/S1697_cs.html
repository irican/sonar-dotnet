当null值判断中的相等运算符或其后的逻辑运算符被反转时，代码看起来会在解引用（dereference）前安全地对其进行null测试。不幸的是，效果恰恰相反——该对象经过测试后，只有当它为null时才会进行解引用，这必然是对null的指针的解引用。

不合规的代码示例

if (str == null && str.Length == 0)
{
  Console.WriteLine("String is empty");
}

if (str != null || str.Length > 0)
{
  Console.WriteLine("String is not empty");
}

合规的解决方案

if (str == null || str.Length == 0)
{
  Console.WriteLine("String is empty");
}

if (str != null && str.Length > 0)
{
  Console.WriteLine("String is not empty");
}

弃用

此规则已弃用，请换用{rule:csharpsquid:S2259}。
