声明却没有使用的局部变量属于死代码（dead code），应该将其删掉以提升代码的可维护性。因为这样开发者就不必搞清这个局部变量的用途。

不合规的代码示例

public int NumberOfMinutes(int hours)
{
  int seconds = 0;   // seconds用不到
  return hours * 60;
}

合规的解决方案

public int NumberOfMinutes(int hours)
{
  return hours * 60;
}

例外

在 using 语句中没有用到的局部创建资源不会报告。

using(var t = new TestTimer()) // t用不到，但这样是合规的。
{
  //……
}
