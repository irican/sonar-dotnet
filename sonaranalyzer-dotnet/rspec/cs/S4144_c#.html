当两个方法具有相同的实现时，不应该无条件地替换集合元素，要么这是一个错误(有其他意图)，要么复制是故意的，但是可能会使维护人员感到困惑。在后一种情况下，一个实现应该调用另一个实现。

不合规的代码示例

private const string CODE = "bounteous";
private int callCount = 0;

public string GetCode()
{
  callCount++;
  return CODE;
}

public string GetName()  // 不合规
{
  callCount++;
  return CODE;
}

合规的解决方案

private const string CODE = "bounteous";
private int callCount = 0;

public string GetCode()
{
  callCount++;
  return CODE;
}

public string GetName()
{
  return GetCode();
}

例外

空方法、只有一行代码的方法和具有相同名称的方法(重载)将被忽略。
