共享命名约定允许团队高效地协作。此规则检查类型名是否使用了帕斯卡命名法（PascalCased）。为了避免误解（reduce noise），两个连续的大写字符只要没有形成完整的类型名，那么就是合规的。因此 MyXClass 是合规的，但 XC 就不合规。

不合规的代码示例

class my_class {...}
class SOMEName42 {...}

合规的解决方案

class MyClass {...}
class SomeName42 {...}

例外

此规则忽略用 ComImportAttribute 或 InterfaceTypeAttribute标记的类型的成员。此外，此规则允许'_' 字符在测试项目（test projects）的类名称中出现

class Some_Name___42 {...} // 在测试（test）中有效
class Some_name___42 {...} // 仍然无效
class Some_Name_XC {...} // 无效 因为出现XC，正确书写应该为 Some_Name_Xc
