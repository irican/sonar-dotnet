当引发事件时，EventHandler 委托期望两个参数：Sender 和 event-data。关于这些参数有三项准则：

  • 在引发非静态事件时，不要将 null 作为发送方传递。
  • 在引发静态事件时，请传递 null 作为发送方。
  • 不要将 null 作为事件数据传递。如果不应该传递数据，那么应该使用 EventArgs.Empty。

当这些指导方针没有得到满足时，这条规则就会引发一个问题。

不合规的代码示例

using System;

namespace MyLibrary
{
  class Foo
  {
    public event EventHandler ThresholdReached;

    protected virtual void OnThresholdReached(EventArgs e)
    {
        ThresholdReached?.Invoke(null, e); // 不合规
    }
  }
}

合规的解决方案

using System;

namespace MyLibrary
{
  class Foo
  {
    public event EventHandler ThresholdReached;

    protected virtual void OnThresholdReached(EventArgs e)
    {
        ThresholdReached?.Invoke(this, e);
    }
  }
}
