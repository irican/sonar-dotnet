常量成员会在编译时复制到调用处，而不是运行时才得到它的值。

例如，假如某个库有一个名为 Version 的常量成员被设为 1.0，并且有一个客户端的应用程序链接到它。然后这个库经过更新，Version 的值被设为了 2.0。不幸的是，即使旧的DLL被新的替换，客户端应用程序的Version 仍然为 1.0 。为了得到 2.0，客户端应用程序需要根据新版本的库进行重建。

这意味你应该用常量来保存那些定义上永远不会改变的值，比如 Zero。但在实际开发中，这种情况并不多见，因此通常来说最好应避免使用常量成员。

此规则只报告公有的常量字段（public constant field），因为他们可以从定义程序集之外访问。

不合规的代码示例

public class Foo
{
    public const double Version = 1.0;           // 不合规
}

合规的解决方案

public class Foo
{
    public static double Version
    {
      get { return 1.0; }
    }
}
