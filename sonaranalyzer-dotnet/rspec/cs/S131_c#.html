结尾必须有 default 从句是出于防御性编程（defensive programming）的考虑。该从句要么实现一个相应的功能，要么包含一条适当的注释以说明此处没有实现功能的原因。甚至，在　switch 语句已经涵盖了一个 enum 类型所有的当前值的情况下，也仍然要使用 default，因为无法保证该 enum 变量不发生扩展。

不合规的代码示例

int foo = 42;
switch (foo) // 不合规
{
  case 0:
    Console.WriteLine("foo = 0");
    break;
  case 42:
    Console.WriteLine("foo = 42");
    break;
}

合规的解决方案

int foo = 42;
switch (foo) // 合规
{
  case 0:
    Console.WriteLine("foo = 0");
    break;
  case 42:
    Console.WriteLine("foo = 42");
    break;
  default:
    throw new InvalidOperationException("Unexpected value foo = " + foo);
}
