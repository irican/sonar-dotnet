如果将一个Windows通讯接口（Windows Communication Foundation）的 OperationContract 方法声明为单向的（one-way），那么该方法不会返回任何结果，甚至连基本的空的确认消息都不会返回。这些方法属于“用后即忘”（fire-and-forget）的方法，在类似于事件（event-like）的通信中很有用。因此对它们指明返回值的类型没有任何意义。

不合规的代码示例

[ServiceContract]
interface IMyService
{
  [OperationContract(IsOneWay = true)]
  int SomethingHappened(int parameter); // 不合规
}

合规的解决方案

[ServiceContract]
interface IMyService
{
  [OperationContract(IsOneWay = true)]
  void SomethingHappened(int parameter);
}

例外

若 OperationContractAttribute.AsyncPattern 被设为 true，则此规则不会报告问题。
