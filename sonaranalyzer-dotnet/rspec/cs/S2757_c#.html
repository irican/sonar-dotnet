=+，=-，=! 这样的成对运算符其实只是将单运算符 +=，-= 和 != 写反了。它们仍然可以通过编译并运行，但你得不到想要的运算结果。

当 =+，=- 或 =! 这样的运算符组合中间没有空格隔开，并且后面至少有一个空格时，此规则会引发一个问题。

不合规的代码示例

int target = -5;
int num = 3;

target =- num;  // 不合规；target = -3，但这是开发者的本意吗？
target =+ num; // 不合规；target = 3

合规的解决方案

int target = -5;
int num = 3;

target = -num;  // 合规；此处显然是要取num的相反数
target += num;
