如果可以将扩展方法添加到类本身，那么创建扩展方法就没有什么意义了。

当扩展在与其扩展的类相同的名称空间中声明时，此规则会引发问题。

不合规的代码示例

namespace MyLibrary
{
    public class Foo
    {
        // ...
    }

    public static class MyExtensions
    {
        public static void Bar(this Foo a) // 不合规
        {
            // ...
        }
    }
}

合规的解决方案

使用单独的命名空间：

namespace MyLibrary
{
    public class Foo
    {
        // ...
    }
}

namespace Helpers
{
    public static class MyExtensions
    {
        public void Bar()
        {
            // ...
        }
    }
}

合并类中的方法：

namespace MyLibrary
{
    public class Foo
    {
        // ...
        public void Bar()
        {
            // ...
        }
    }
}
