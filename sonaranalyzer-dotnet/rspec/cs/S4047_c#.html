当使用引用参数(关键字ref)时，传递的参数类型必须与引用参数类型完全匹配。这意味着要传递派生类型，必须强制转换并将其分配给适当类型的变量。通用方法的使用消除了向下强制转换的麻烦，因此应该是首选方法。

当方法包含类型为System.Object的ref参数时，此规则会引发一个问题。

不合规的代码示例

using System;

namespace MyLibrary
{
  public class Foo
  {
    public void Bar(ref object o1, ref object o2) // 不合规
    {
    }
  }
}

合规的解决方案

using System;

namespace MyLibrary
{
  public class Foo
  {
    public void Bar<T>(ref T ref1, ref T ref2)
    {
    }
  }
}
