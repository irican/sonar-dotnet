如果你无法提供某个对象的所有内容（flesh the object fully），在构造函数里就抛出异常可能会是个好办法，但不要在 exception 的构造函数里抛出。如果这样做了，你会干扰到最初抛出的异常。另外，在创建异常时抛出的异常很可能不会在调用代码中得到正确的处理，并且，意外的、未处理的异常会导致程序的终结。

不合规的代码示例

class MyException: Exception
{
    public void MyException()
    {
         if (bad_thing)
         {
             throw new Exception("A bad thing happened");  // 不合规
          }
    }
}
