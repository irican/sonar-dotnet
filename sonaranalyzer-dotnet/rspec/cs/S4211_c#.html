透明度属性、SecurityCriticalAttribute 和 SecuritySafeCriticalAttribute 用于标识执行安全关键操作的代码。第二种方法表明从透明中调用这段代码是安全的，而第一种方法则不安全。由于具有较大范围的代码元素的透明属性优先于包含在第一个元素a类中的代码元素的透明属性，例如，具有SecurityCriticalAttribute 的代码不能包含具有SecuritySafeCriticalAttribute的方法。

当使用 System.Security 安全属性标记成员时，此规则会引发一个问题，该属性的透明度与成员容器的安全属性不同。

不合规的代码示例

using System;
using System.Security;

namespace MyLibrary
{

    [SecurityCritical]
    public class Foo
    {
        [SecuritySafeCritical] // 不合规
        public void Bar()
        {
        }
    }
}

合规的解决方案

using System;
using System.Security;

namespace MyLibrary
{

    [SecurityCritical]
    public class Foo
    {
        public void Bar()
        {
        }
    }
}
