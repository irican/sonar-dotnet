只有setter的属性是不符合人们的使用习惯的，会让人感到困惑。相反地，如果可能的话应添加一个getter，或者用setter方法替换该属性。

不合规的代码示例

class Program
{
    public int Foo  //不合规
    {
        set
        {
            // ……一些代码……
        }
    }
}

合规的解决方案

class Program
{
    private int foo;

    public void SetFoo(int value)
    {
        // ……一些代码……
        foo = value;
    }
}

或者

class Program
{
  public int Foo { get; set; } // 合规
}
