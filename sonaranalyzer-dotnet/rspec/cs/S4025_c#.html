子类中的字段与父类的字段的名称仅在大小写上不同，这肯定会引起混淆。这样的子类字段应该重新命名。

不合规的代码示例

public class Fruit
{
  protected string plantingSeason;
  //...
}

public class Raspberry : Fruit
{
  protected string plantingseason;  // 不合规
  // ...
}

合规的解决方案

public class Fruit
{
  protected string plantingSeason;
  //...
}

public class Raspberry : Fruit
{
  protected string whenToPlant;
  // ...
}

或者

public class Fruit
{
  protected string plantingSeason;
  //...
}

public class Raspberry : Fruit
{
  // 字段删除；将改用父子段
  // ...
}
