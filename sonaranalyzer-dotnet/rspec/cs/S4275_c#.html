属性通过提供 public，protected 或 internal方法来提供对 private 字段的受控访问，从而提供了一种强制封装的方法。然而，在具有多个字段的类中，使用剪切和粘贴来快速创建所需的属性并不少见，这可能导致getter或setter访问了错误的字段。

这条规则在以下任何情况下都会引起问题：

  • setter不使用相应的名称更新字段。
  • getter不访问具有对应名称的字段。

对于简单的属性，最好使用 自动实现的属性 (C# 3.0 或之后的版本)。

不合规的代码示例

class A
{
    private int x;
    private int y;

    public int X
    {
        get { return x; }
        set { x = value; }
    }

    public int Y
    {
        get { return x; }  // 不合规: 返回值中不使用字段 'y' 
        set { x = value; } // 不合规: 字段'y'没有更新
    }
}

合规的解决方案

class A
{
    private int x;
    private int y;

    public int X
    {
        get { return x; }
        set { x = value; }
    }

    public int Y
    {
        get { return y; }
        set { y = value; }
    }
}
