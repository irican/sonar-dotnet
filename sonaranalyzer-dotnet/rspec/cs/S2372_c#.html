属性的getter方法应该是可安全调用的简单操作。如果需要抛出异常，最好将属性转换为方法。

可以从属性的setter方法和被编入索引的属性所包含的getter方法中抛出异常，此规则并不会对此进行检查。

不合规的代码示例

public int Foo
{
    get
    {
        throw new Exception(); // 不合规
    }
}

合规的解决方案

public int Foo
{
    get
    {
        return 42;
    }
}

例外

抛出的异常如果是继承自或本身属于 NotImplementedException， NotSupportedException 或 InvalidOperationException 时不会引发问题。
