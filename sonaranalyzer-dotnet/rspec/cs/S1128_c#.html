即使没有必要的 using 语句不会对生成的应用有任何改变，去掉它们会：

  • 提高可读性和可维护性。
  • 减少编码时集成编辑环境（IDE）自动补全的提示数量。
  • 可能避免命名冲突。
  • 由于解析类型时所能寻找的命名空间变少，编译时间会增加。
  • 命名空间从项目中被移除，构筑可能会失败。

不合规的代码示例

using System.Collections.Generic; // 不合规 - 不必要的using

namespace Foo
{
    public class Bar
    {
        public Bar(string path)
        {
            File.ReadAllLines(path);
        }
    }
}

合规的解决方案

using System.IO;

namespace Foo
{
    public class Bar
    {
        public Bar(string path)
        {
            File.ReadAllLines(path);
        }
    }
}
