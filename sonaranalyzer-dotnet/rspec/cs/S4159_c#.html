在属性化编程模型中，ExportAttribute 声明了一个部件“导出”，或者向组合容器提供了一个对象，以满足特定的契约。在组合期间，具有匹配契约的导入部分的依赖项将由导出对象填充。

如果类型没有实现它导出的接口，那么在运行时就会出现问题(要么是强制转换异常，要么只是容器没有填充导出的类型)，从而导致意外的行为/崩溃。

当类没有实现或继承ExportAttribute中声明的类型时，该规则会引发一个问题。

不合规的代码示例

[Export(typeof(ISomeType))]
public class SomeType // 不合规；没有实现'ISomeType'。
{
}

合规的解决方案

[Export(typeof(ISomeType))]
public class SomeType : ISomeType
{
}
