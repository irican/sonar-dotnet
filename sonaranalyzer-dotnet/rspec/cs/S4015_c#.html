将继承的成员更改为private不会阻止对基类实现的访问。

当未密封类型中的private方法具有与基类型中声明的public方法相同的签名时，此规则会引发一个问题。

不合规的代码示例

using System;

namespace MyLibrary
{
  public class Foo
  {
    public void SomeMethod(int count) { }
  }
  public class Bar:Foo
  {
    private void SomeMethod(int count) { } // 不合规
  }
}

合规的解决方案

using System;

namespace MyLibrary
{
  public class Foo
  {
    public void SomeMethod(int count) { }
  }
  public sealed class Bar : Foo
  {
    private void SomeMethod(int count) { }
  }
}
